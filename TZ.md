# Техническое Задание (ТЗ): Сервис совместного планирования «ScheduleHelper»

## 1. Общее описание
«ScheduleHelper» — это легковесный web-сервис для командного поиска общего времени и назначения встреч. Пользователи отмечают своё «свободное время» в личном календаре, а на странице общего календаря формируется единая картина доступности всей команды, поверх которой можно создавать общие события.

## 2. Архитектура и технологический стек
- **Бэкенд:** `Golang` (Go) + встроенная БД `SQLite`. Гарантирует минимальное потребление ОЗУ и CPU, работает как единый бинарный файл.
- **Фронтенд:** `Svelte`. Обеспечивает высокую скорость отрисовки, компилируется в чистый JS без тяжелого виртуального DOM. Элегантный UI на чистом CSS (без громоздких фреймворков).
- **Инфраструктура:** Docker-контейнер для хостинга на слабом железе. Запуск через `docker compose up -d --build`.
- **Синхронизация (Real-time):** Изменения (создание/изменение слотов и событий) автоматически сохраняются и транслируются другим пользователям без перезагрузки страницы (допускаются небольшие задержки, реализуется через WebSockets или SSE).
- **Архитектурные закладки (на будущее):** На уровне БД и API закладывается возможность авторизации по паролю и интеграции/синхронизации данных с Google-календарем.

## 3. Авторизация и профиль пользователя
- **Визуальное представление профиля (Компонент «Профиль»):**
  - Используется везде, где нужно отобразить пользователя (экран входа, шапка, списки).
  - Состоит из круга с градиентом выбранного пользователем цвета.
  - Внутри круга располагается выбранный эмоджи-аватар.
  - Справа от круга отображается имя пользователя, написанное выбранным цветом.
  - Для обеспечения читаемости имени на любом фоне (светлом или тёмном) применяется контрастное обрамление (белая или черная обводка/подложка текста).

- **Экран входа:** До авторизации применяется тема оформления из ОС пользователя (системная). Вход основан на доверии.
  - На странице отображается список всех зарегистрированных пользователей в виде сетки или списка с переносами.
  - Каждый элемент списка — это полный компонент «Профиль» (Круг с градиентом, эмоджи и имя справа).
  - Пользователь находит свой профиль в списке и кликает/тапает по нему для входа.
  - Если пользователя нет в списке, он нажимает кнопку «Зарегистрироваться» (или «Добавить профиль»).

- **Регистрация:** Отдельная форма создания нового профиля, где пользователь задает:
  1. Эмоджи-аватар (при тапе выбор из 40-50 вариантов: животные, эмоции, предметы вроде сердечка/луны).
  2. Индивидуальный цвет (используется для градиента круга и цвета имени).
  3. Уникальное имя (проверка на уникальность обязательна).
- **Настройки профиля:**
  - Открываются по тапу/клику на иконку или имя пользователя в правой части шапки сайта.
  - Представляют собой плавающую форму, где элементы расположены сверху вниз:
    - Изменение эмоджи, цвета, имени.
    - Настройки часового пояса: галочка "Определять автоматически" и выпадающий список для ручного выбора.
    - Выбор темы оформления: Светлая, Тёмная, Системная.
    - Кнопка "Удалить профиль" (удаление возможно только из-под своей учетной записи).

## 4. Глобальные правила
- **Часовые пояса:** Все временные отрезки и события переводятся и отображаются в локальном часовом поясе пользователя, который смотрит на экран.
- **Очистка данных:** На сервере раз в неделю запускается фоновый процесс (cron), который удаляет прошедшие слоты и события старше 6 месяцев для экономии места и ресурсов.

## 5. Страница 1: Общий календарь (Главная вкладка)
- **Назначение:** Просмотр доступности команды и создание общих событий.
- **Таблица участников (Ось Y):**
  - Отображаются **только** те пользователи, у которых есть свободные отрезки в просматриваемом периоде.
  - **Имена пользователей:** Окрашены в выбранный ими цвет. Для обеспечения читаемости на любом фоне используется контрастное обрамление (белая или черная обводка/подложка текста).
  - **Сортировка:** Поддерживается глобальная сортировка перетаскиванием (drag-and-drop). Измененный порядок применяется для всех пользователей сервиса.
- **Сетка времени (Ось X):**
  - Десктоп: по умолчанию 10 дней.
  - Мобильные: по умолчанию 4 дня.
  - Плавный горизонтальный скролл в будущее и прошлое. Элегантные, ненавязчивые разделители ячеек.
- **Отрезки времени (Слоты):**
  - Окрашены в цвет пользователя.
  - При наведении (десктоп) или тапе (мобильные) показывается точное время и **комментарий пользователя** (если он был оставлен к этому слоту).
- **Общие события (Events):**
  - **Визуализация:** Вертикальные полупрозрачные блоки, проходящие сверху вниз через все строки пользователей.
  - **Шапка события (над датами):** 
    - Строго **Иконка** сверху (если указана для события), под ней **Заголовок**. 
    - Ограничение заголовка — 120 символов. Если текст не влезает по ширине колонки, он обрезается многоточием (показывается вмещаемая часть).
    - Цветовая палитра событий: оттенки золотого (напоминающие золотой градиент).
  - **Создание:** Клик/тап по области дат или пустой области заголовка/иконки открывает плавающую форму:
    1) Иконка, 2) Оттенок золотого, 3) Заголовок (до 120 симв.), 4) Описание (с поддержкой ссылок), 5) Точные дата и время начала и завершения события.
  - **Взаимодействие:** При наведении на иконку/заголовок всплывает карточка с полным нередактируемым описанием (ссылки кликабельны, текст можно выделять). В таблице **подсвечиваются участники**, чьи отрезки времени пересекаются с этим событием.

## 6. Страница 2: Личный календарь
- **Назначение:** Отметка своего свободного времени.
- **Интерфейс таблицы:**
  - Строки — дни (текущий и будущие).
  - Столбцы — часы внутри дня.
  - Локальные настройки на странице: количество отображаемых дней (по умолчанию 14) и видимый временной диапазон (по умолчанию 07:00 – 00:00).
- **Создание слотов:**
  - Десктоп: Клик, удержание и перетаскивание (слайд) курсора.
  - Мобильные: Тап по началу и проведение пальцем (слайд) до конца.
  - Точность привязки (шаг) — 30 минут.
- **Редактирование слотов:** Клик/тап по существующему отрезку открывает меню:
  1. Ввод точного времени вручную (до минут).
  2. Разрыв отрезка ровно в месте клика на две независимые части.
  3. **Добавление/редактирование комментария** к отрезку.
- **Отмена действий:** Плавающая кнопка (иконка сбоку) «Отмена» (Undo) для отмены последних локальных действий (которые еще в памяти браузера).

## 7. UI/UX Детали
- Поддержка двух тем (Светлая / Тёмная) with плавным переключением.
- Отсутствие вычурных эффектов, акцент на легкость восприятия, скругленные углы, аккуратные тени (Material/Apple-like минимализм).
- Переключение между страницами "Общий календарь" и "Личный календарь" через табы в верхней части страницы (под шапкой).
