# Objective
Реализовать web-сервис совместного планирования «ScheduleHelper» согласно утвержденному `TZ.md`. 
Сервис будет включать Go-бэкенд с SQLite, Svelte-фронтенд, и будет запакован в Docker-контейнер. Основная задача: пошаговая реализация с покрытием тестами, автоматическим прогоном регрессии и фиксацией стабильных этапов в Git.

# Key Files & Context
- `C:\sources\AI\ScheduleHelper\TZ.md` — Основное техническое задание.
- `backend/` — Директория для Go (API, SQLite, cron, WebSocket/SSE).
- `frontend/` — Директория для Svelte (UI, логика, стейт, стили).
- `Dockerfile` и `docker-compose.yml` — Инфраструктура.

# Implementation Steps

## Epic 1: Инициализация проекта и инфраструктуры
1. **Инициализация репозитория Git**.
2. **Настройка бэкенда**: Создание Go-модуля (`go mod init`), базовая структура (handlers, models, db).
3. **Настройка фронтенда**: Инициализация Svelte-проекта (через Vite).
4. **Настройка Docker**: Создание `docker-compose.yml` (`docker compose up -d --build`) и `Dockerfile` (multi-stage build для Go и Svelte).
5. Написание базового smoke-теста (проверка доступности сервиса). Первый git-коммит.

## Epic 2: Бэкенд — База данных, Пользователи и Профиль
1. **Проектирование БД**: Создание схемы SQLite (таблицы: `users`, `slots`, `events`, `event_participants`). Учесть закладки под пароли и Google Calendar.
2. **API Пользователей**: Эндпоинт для получения списка всех пользователей (`GET /api/users`) для экрана входа.
3. **API Авторизации**: Эндпоинты для входа (`POST /api/login`) и регистрации (`POST /api/register`). Проверка уникальности имени при регистрации.
4. **API Профиля**: Эндпоинты для получения и обновления настроек профиля (тема, timezone, цвет, эмоджи), а также удаления профиля.
5. Покрытие unit-тестами Go-хэндлеров и БД-операций. Прогон регрессии. Git-коммит.

## Epic 3: Бэкенд — Слоты и Общие события
1. **API Слотов**: CRUD для личных свободных отрезков времени (с учетом комментариев).
2. **API Событий**: CRUD для общих событий (заголовок, описание, даты, иконка, цвет).
3. **API Общего календаря**: Эндпоинт для получения агрегированных данных о доступности пользователей на заданный период.
4. **Очистка данных**: Настройка фонового процесса (cron) для очистки данных старше 6 месяцев (каждую неделю).
5. Покрытие unit-тестами. Прогон регрессии. Git-коммит.

## Epic 4: Фронтенд — Архитектура, Базовый UI и Авторизация
1. **Роутинг и стейт**: Настройка stores в Svelte для пользователя, списка пользователей, темы, таймзоны.
2. **Компонент «Профиль»**: Реализация переиспользуемого компонента (Круг с градиентом + эмоджи + имя выбранным цветом с контрастной обводкой).
3. **Экран входа**: Отображение сетки существующих пользователей (каждый элемент — компонент «Профиль») для быстрого входа и кнопка «Зарегистрироваться».
4. **Форма регистрации**: Отдельный экран/модал для создания профиля с выбором аватара (40-50 эмоджи), цвета и ввода имени.
5. **Лейаут и Шапка**: Создание шапки с табами (Общий/Личный календарь) и доступом к плавающей форме профиля.
5. Написание UI-тестов. Прогон регрессии. Git-коммит.

## Epic 5: Фронтенд — Личный календарь
1. **Таблица расписания**: Верстка сетки дней и часов (07:00-00:00 с возможностью изменения).
2. **Интерактив (Слоты)**: Реализация drag-and-drop / тап-свайп для рисования слотов (шаг 30 минут).
3. **Редактирование**: Клик/тап для ручного ввода времени, разрыва слота, редактирования комментария.
4. **Отмена (Undo)**: Реализация локальной истории действий и кнопки отмены.
5. Интеграция с API (сохранение на лету). UI-тестов. Прогон регрессии. Git-коммит.

## Epic 6: Фронтенд — Общий календарь
1. **Сетка и Участники**: Отображение только активных участников с их слотами (цвета, контрастные имена). Плавный горизонтальный скролл.
2. **Сортировка**: Drag-and-drop сортировка списка участников. Сохранение порядка глобально.
3. **Общие события**: Отрисовка вертикальных полупрозрачных блоков, шапки (иконка над заголовком с "золотой" палитрой).
4. **Форма события**: Плавающая форма создания/редактирования, карточка просмотра (тултип) с подсветкой пересекающихся участников.
5. UI-тестов. Прогон регрессии. Git-коммит.

## Epic 7: Real-Time Синхронизация
1. **WebSockets / SSE**: Добавление транспорта на бэкенд для трансляции изменений (создание слота/события, изменение сортировки).
2. **Интеграция на фронтенде**: Подписка на события и реактивное обновление UI (stores).
3. Интеграционные тесты. Прогон регрессии. Git-коммит.

# Verification & Testing
- Unit-тесты для бизнес-логики Go (база данных, сервисы).
- Unit-тесты для Svelte компонентов (проверка рендера, стейт-менеджмента).
- Smoke-тесты запуска Docker-контейнера (`docker compose up -d --build`).
- Регрессионные тесты после каждого эпика/фичи.
- Все тесты будут запускаться перед созданием очередного git-коммита.
